 هر نود را به یک «نوت Zettel واقعی» تبدیل کن
برای هر نود از [[DAG]]:
یک فایل/نوت جدا بساز
عنوان = نام نود
محتوا = فقط یک ایده
یک مثال کوتاه یا یک جمله‌ی تعریف‌کننده (حداکثر 2 خط)
قانون طلایی:
اگر نوتت 5 خط بیشتر شد، یعنی باید تقسیمش کنی.

___
 برای هر نود یک «سؤال یادگیری» بساز
این بخش خیلی مهمه.
هر نود باید یک سؤال برای «تست فهم» داشته باشد.
مثلاً:
> Workflow as State Machine → “چرا workflow باید state machine باشد؟”
> Transactions → “بدون transaction چه race conditionهایی رخ می‌دهد؟”
> Sharding → “چرا sharding برای مقیاس‌پذیری لازم است؟”
> Local Transfer Queue → “چطور transfer queue تراکنش را حفظ می‌کند؟”
این‌ها می‌شوند فلش‌کارت‌های ذهنی.


___

 هر نود را به یک «تمرین کوچک» تبدیل کن
تمرین‌ها باید واقعی و عملی باشند.
مثلاً برای نود Transactions:
تمرین: یک سیستم ساده با DB + Queue بساز و نشان بده که بدون transaction چه race conditionی رخ می‌دهد.
یا برای Sharding:
تمرین: فرض کن 10 shard داری، workflow id را hash کن و shard allocation را طراحی کن.
این‌ها را می‌توانی به عنوان «پروژه کوچک» ذخیره کنی.
